\begin{frame}
  \frametitle{Binding Library}
  \begin{itemize}
  \item In most cases one library suffices
  \item The default library name is \myLstinline{CGALPY}
  \item The bindings for distinct instances of a templace must be in separate
    libraries
  \item The \cmake{} flag \cmakeLstinline{CGALPY_FIXED_LIBRARY_NAME} determines
    whether the name of the generated library is \myLstinline{CGALPY} or not
  \item For example, the library named
    \myLstinline{GALPY_kernelEpecInt_Aos2SegPlainPl} contains bindings for
    \begin{itemize}
      \item the EPEC Kernel and intersections, and
      \item the instance of \cppLstinline{Arrangement_2<GeometryTraits, Dcel>}
        class template, where
        \begin{itemize}
        \item \cppLstinline{GeometryTraits} is substituted with
          an instance of the traits class template that handles segments, and
        \item \cppLstinline{Dcel} is substituted with the default DCEL
        \end{itemize}
    \end{itemize}
  \end{itemize}
\end{frame}
